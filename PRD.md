# Polymarket 套利筛选器 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
Polymarket 套利筛选器

### 1.2 产品定位
一个移动端优先的预测市场筛选工具，帮助用户快速发现具有认知判断价值的 Polymarket 项目，过滤掉纯随机性事件，为套利决策提供支持。

### 1.3 目标用户
- 预测市场交易者
- 套利投资者
- 对政治、经济、科技等领域有认知优势的分析师

### 1.4 核心价值
- **节省时间**: 自动过滤无价值的随机性项目
- **发现机会**: 按交易量排序，快速定位高流动性市场
- **降低门槛**: 中文翻译，降低英语阅读障碍
- **移动友好**: 随时随地查看和筛选

## 2. 产品目标

### 2.1 业务目标
| 目标 | 指标 | 说明 |
|------|------|------|
| 提高筛选效率 | 过滤率 > 80% | 过滤掉随机性项目 |
| 发现高价值项目 | Top 10 平均交易量 > $1M | 优先展示高流动性项目 |
| 用户留存 | 收藏功能使用率 | 用户愿意收藏感兴趣项目 |

### 2.2 用户目标
- 快速浏览有价值的预测市场
- 识别具有信息优势的套利机会
- 跟踪感兴趣的项目

## 3. 功能需求

### 3.1 核心功能

#### F1: 项目筛选
| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 自动过滤随机性项目，只展示认知判断类项目 |

**过滤规则:**
- 标题包含 "Up or Down", "15m", "1h", "4h" 等关键词
- 标签包含 "up-or-down", "crypto-prices", "recurring"
- 交易量为0且赔率为50-50的未开始项目

**验收标准:**
- [ ] 加密货币价格涨跌预测被过滤
- [ ] 15分钟/1小时周期事件被过滤
- [ ] 政治、经济、科技类项目正常显示

#### F2: 赔率显示
| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 显示每个项目的 Yes/No 赔率 |

**显示规则:**
- Yes 赔率: 绿色显示
- No 赔率: 红色显示
- 格式: XX¢ (0-100)

**验收标准:**
- [ ] 赔率数据正确解析
- [ ] 颜色区分明确
- [ ] 非50-50赔率的项目优先显示

#### F3: 交易量/流动性显示
| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 显示每个项目的交易量和流动性 |

**显示规则:**
- 格式化显示: $1.2M, $50K
- 按交易量降序排列

**验收标准:**
- [ ] 交易量数据正确显示
- [ ] 流动性数据正确显示
- [ ] 高交易量项目排在前面

#### F4: 价值评分
| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 根据交易量和流动性计算价值分数 |

**评分规则:**
| 条件 | 分数 | 颜色等级 |
|------|------|----------|
| Vol > $500K 或 Liq > $100K | 90 | 高 (绿色) |
| Vol > $100K 或 Liq > $50K | 75 | 高 (绿色) |
| Vol > $50K 或 Liq > $20K | 60 | 中 (黄色) |
| Vol > $10K 或 Liq > $5K | 45 | 中 (黄色) |
| 其他 | 30 | 低 (红色) |

**验收标准:**
- [ ] 分数计算正确
- [ ] 卡片边框颜色与等级对应
- [ ] 进度条正确显示分数

#### F5: 中文翻译
| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 自动翻译标题为中文 |

**实现方式:**
- 使用 MyMemory 免费翻译 API
- 异步翻译，不阻塞页面加载
- 翻译结果缓存到 localStorage

**验收标准:**
- [ ] 标题翻译为中文
- [ ] 原文显示在翻译下方
- [ ] 翻译失败时显示原文

#### F6: 收藏功能
| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 用户可以收藏感兴趣的项目 |

**功能点:**
- 点击星标收藏/取消收藏
- 收藏列表独立展示
- 数据存储在 localStorage

**验收标准:**
- [ ] 收藏状态正确保存
- [ ] 收藏列表正确显示
- [ ] 刷新页面后收藏状态保持

#### F7: 删除功能
| 属性 | 说明 |
|------|------|
| 优先级 | P2 |
| 描述 | 用户可以删除不感兴趣的项目 |

**功能点:**
- 点击X删除卡片
- 删除后滑出动画
- 删除记录持久化

**验收标准:**
- [ ] 删除动画流畅
- [ ] 删除后不再显示
- [ ] 刷新页面后删除状态保持

#### F8: 瀑布流加载
| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 分批加载项目，支持无限滚动 |

**实现方式:**
- 每次获取100个，过滤后显示10个
- 点击"加载更多"获取下一批
- 自动过滤已显示项目

**验收标准:**
- [ ] 首次加载显示10个项目
- [ ] 点击加载更多获取下一批
- [ ] 加载状态正确显示

### 3.2 辅助功能

#### F9: 项目详情跳转

#### F10: 结束时间显示

## 6. 数据流

### 6.1 数据获取流程
```
用户打开页面
    ↓
请求 /api/events (通过 Nginx 代理)
    ↓
Nginx 转发到 gamma-api.polymarket.com
    ↓
返回事件列表 (100个)
    ↓
前端过滤随机性项目
    ↓
按交易量排序
    ↓
显示前10个
    ↓
异步翻译标题
```

### 6.2 本地存储结构
```javascript
// 收藏 ID 列表
localStorage['pm_favorites'] = '["event_id_1", "event_id_2"]'

// 收藏数据
localStorage['pm_fav_data'] = '{"event_id_1": {...}, "event_id_2": {...}}'

// 删除 ID 列表
localStorage['pm_deleted'] = '["event_id_3"]'

// 翻译缓存
localStorage['pm_trans'] = '{"t_event_id_1": "中文标题"}'
```
